# Apple M4 GPU — Measured Properties

All numbers from real probe runs on this device.

## Device Info
- **Device:** Apple M4, unified memory
- **Threads per SIMD group:** 32
- **Max threadgroup size:** 1024 threads
- **Max shared memory per threadgroup:** 32 KB
- **Max buffer size:** 9093 MB
- **Recommended working set:** 12124 MB
- **Kernel launch overhead:** ~1.5 us
- **Smallest measurable GPU time:** ~2.6 us

## Memory Bandwidth
- **Main memory (DRAM):** ~102 GB/s
- **Shared memory (threadgroup):** ~3215 GB/s aggregate
- **Texture reads:** ~479 GB/s
- **Buffer reads:** ~764 GB/s

## Cache Hierarchy
Load latency at each working set size:

- **4 KB:** ~43 ns
- **16 KB:** ~43 ns
- **64 KB:** ~44 ns
- **256 KB:** ~58 ns
- **1024 KB:** ~89 ns
- **4096 KB:** ~133 ns
- **16384 KB:** ~188 ns
- **32768 KB:** ~201 ns
- **System-level cache associativity:** 16-way
- **Shared memory bank count:** 32

## Memory Access Patterns
- **Sequential (coalesced) reads:** ~147 GB/s
- **Strided (non-coalesced, stride 16) reads:** ~124 GB/s
- ~1.2x penalty for scattered access

## Compute Throughput
- **32-bit float:** ~5170 GFLOPS
- **16-bit float:** ~5169 GFLOPS (same speed as 32-bit — no packed fp16 execution)
- **32-bit integer:** ~971 billion ops/s (~5.3x slower than float)
- **Shader core count:** ~31 (inferred from throughput saturation)

## Cross-Thread Communication
- **SIMD shuffle (lane swap):** ~833 billion/s, ~3334 GB/s effective
- **SIMD reduction (sum across lanes):** ~115 billion/s

## Atomic Operations
- **No contention (many counters):** ~26.2 billion ops/s
- **Full contention (single counter):** ~0.8 billion ops/s
- Contention penalty: ~33x slowdown

## Synchronization Costs
- **Barrier cost (32 threads):** ~19 ns
- **Barrier cost (1024 threads):** ~69 ns
- **Shared memory write-then-read latency (1 thread):** ~55 ns
- **Shared memory write-then-read latency (256 threads):** ~108 ns

## Register File
- **Spill boundary:** between 24 and 32 matrix accumulators (8x8 float tiles)
- Beyond this, the compiler spills to memory and performance collapses.

## Execution Pipeline
- **Peak throughput (independent ops):** ~1230 GFLOPS
- **Latency-limited (dependent chain):** ~646 GFLOPS
- Need ~2 independent operations in flight to fully utilize the pipeline

## Instruction Cache
- **Small kernel (4 ops/loop):** ~1291 GFLOPS
- **Medium kernel (32 ops/loop):** ~2790 GFLOPS
- **Large kernel (64 ops/loop):** ~2033 GFLOPS
- Large kernel drops ~27% — instruction cache can't hold the full loop body

## Address Translation (TLB)
Latency per load when touching N distinct memory pages (16 KB each):

- **4 pages:** ~43 ns
- **8 pages:** ~43 ns
- **16 pages:** ~43 ns
- **32 pages:** ~43 ns
- **64 pages:** ~43 ns
- **128 pages:** ~43 ns
- **256 pages:** ~43 ns
- **512 pages:** ~44 ns
- **1024 pages:** ~45 ns

## Matrix Multiply Unit
- **Single dependent multiply-accumulate:** ~0.3 ns each
- **Four independent multiply-accumulates:** ~0.2 ns each
- ~2.0x speedup when the GPU can overlap independent matrix ops

## Kernel Launch Overhead
- **Direct launch:** ~1.4 us
- **Indirect launch (grid size from GPU buffer):** ~2.9 us
- **Indirect overhead:** ~1.5 us extra

## GPU-to-CPU Readback
| Size | GPU Time | Readback Time | Total |
|------|----------|---------------|-------|
| 4 KB | 1 us | 182 us | 184 us |
| 64 KB | 2 us | 171 us | 173 us |
| 1024 KB | 5 us | 174 us | 180 us |
| 16384 KB | 123 us | 230 us | 353 us |

## Concurrent Kernel Execution
- **Kernel A alone:** 1.275 ms
- **Kernel B alone:** 1.272 ms
- **Both in same command buffer:** 2.550 ms
- **Overlap:** -0%
- The GPU serialized them — no concurrent execution observed.

## Dynamic Cache Partitioning (M3+ only)
Tests whether the chip dynamically rebalances on-chip memory between registers and shared memory.

| Configuration | Trillion ops/s |
|---------------|----------------|
| Low shared mem (256B) + 16 accumulators | 2.581 |
| Medium shared mem (8KB) + 16 accumulators | 2.606 |
| High shared mem (24KB) + 16 accumulators | 2.602 |

Minimal drop — chip dynamically rebalances (M3+ behavior).

## BFloat16 Support
- **BFloat16 (half-precision proxy):** ~6079 GFLOPS
- **Float16:** ~5169 GFLOPS
- Same speed — no dedicated bfloat16 hardware, runs on the float16 pipe.

## Atomic Operation Variants
- **32-bit integer atomic (native hardware):** ~24.6 billion ops/s
- **32-bit float atomic (compare-and-swap loop):** ~0.4 billion ops/s
- **64-bit atomic (emulated with 32-bit):** ~16.8 billion ops/s
- Float atomic is 56x slower — no native float atomic support, falls back to software loop.
